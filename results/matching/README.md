# 匹配结果目录说明

## 目录结构

每次运行匹配流程都会创建一个独立的时间戳文件夹，格式为：`run_YYYYMMDD_HHMMSS`

### 文件夹命名规则
- `run_20260225_191017` - 表示2026年2月25日19点10分17秒的运行结果

### 每个run文件夹包含以下文件：

#### 1. `match_pairs_TIMESTAMP.txt`
- 匹配对的详细列表
- 按相似度排序的碎片匹配关系

#### 2. `fragment_matches_TIMESTAMP.json`
- JSON格式的碎片匹配详情
- 包含每个碎片的所有匹配对信息

#### 3. `fragment_matches_TIMESTAMP.txt`
- 人类可读的碎片匹配列表
- 按碎片组织的匹配结果

#### 4. `matching_report_TIMESTAMP.md`
- 完整的匹配报告
- 包含统计信息、匹配分布、详细结果等

#### 5. `process_details_TIMESTAMP.json`
- 匹配过程的详细技术信息
- 包含参数设置、中间计算结果等

## 使用建议

- 每次运行都会自动创建新的时间戳文件夹
- 建议定期清理过时的运行记录以节省空间
- 可以通过查看`matching_report_*.md`文件快速了解每次运行的主要结果
- `process_details_*.json`文件适合深入分析技术细节

## 注意事项

- 不要手动修改run文件夹中的文件名
- 如需比较不同运行结果，建议复制相应的run文件夹到其他位置进行分析
- 清理旧的run文件夹时，请确保备份重要结果
